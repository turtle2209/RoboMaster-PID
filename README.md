## :book: 有关前馈PID的个人见解
  &emsp;	&emsp;	云台带一定负载的时候，它想从平视抬头到一个目标角度，一般来说理论上他是可以到的，p项可以让他到这个目标角度，但是却维持不了，因为维持是需要积分项i累积的，但是这个累积的过程又需要时间，所以这时候会出现一个现象就是先抬头到目标角度，然后又低头一点，然后又慢慢抬头到目标角度，但是此时是积分项i在作用，也就意味着i会一直积分，如果系统要到下一个新的目标角度的时候，需要花同样的时间抵消这个积分，导致系统迟钝。

 &emsp;	&emsp;	 而前馈，就是 基础PID算式+ k_前馈*前馈值 ，前馈值一般是设置的目标角度，也就是说基础pid是事后诸葛亮，等误差发生了才有反应，而前馈pid是通过你要达到的目标值再乘上一个系数来放大这个值，新的目标值一设定好，他就会直接输出来抵消重力作用，也就是说它不是依靠误差值来输出动力的。他是通过目标值成一个放大系数来输出动力。

## :book: 有关仿真的个人见解
 &emsp;	&emsp;仿真和现实物理存在一定的差别，仿真调出来的pid不一定就可以直接用于物理世界，采样频率不一样，仿真一般是连续的，而在单片机上跑的是离散的数据。不过仿真可以看物理规律，可以看pid调参的大方向，对于更复杂的系统来说，仿真奠定了调参大方向，减小了试错成本。
